name: Mobile Code
text: "|-\n  a. Define acceptable and unacceptable mobile code and mobile code technologies;\
  \ and\n  b. Authorize, monitor, and control the use of mobile code within the system."
discussion: Mobile code includes any program, application, or content that can be
  transmitted across a network (e.g., embedded in an email, document, or website)
  and executed on a remote system. Decisions regarding the use of mobile code within
  organizational systems are based on the potential for the code to cause damage to
  the systems if used maliciously. Mobile code technologies include Java applets,
  JavaScript, HTML5, WebGL, and VBScript. Usage restrictions and implementation guidelines
  apply to both the selection and use of mobile code installed on servers and mobile
  code downloaded and executed on individual workstations and devices, including notebook
  computers and smart phones. Mobile code policy and procedures address specific actions
  taken to prevent the development, acquisition, and introduction of unacceptable
  mobile code within organizational systems, including requiring mobile code to be
  digitally signed by a trusted source.
