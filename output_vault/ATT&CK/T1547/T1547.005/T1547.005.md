---
title: 'Boot or Logon Autostart Execution: Security Support Provider'
desc: "|-\n  Adversaries may abuse security support providers (SSPs) to execute DLLs\
  \ when the system boots. Windows SSP DLLs are loaded into the Local Security Authority\
  \ (LSA) process at system start. Once loaded into the LSA, SSP DLLs have access\
  \ to encrypted and plaintext passwords that are stored in Windows, such as any logged-on\
  \ user's Domain password or smart card PINs.\n  \n  The SSP configuration is stored\
  \ in two Registry keys: <code>HKLM\\SYSTEM\\CurrentControlSet\\Control\\Lsa\\Security\
  \ Packages</code> and <code>HKLM\\SYSTEM\\CurrentControlSet\\Control\\Lsa\\OSConfig\\\
  Security Packages</code>. An adversary may modify these Registry keys to add new\
  \ SSPs, which will be loaded the next time the system boots, or when the AddSecurityPackage\
  \ Windows API function is called.(Citation: Graeber 2014)"
related_frameworks:
- '[[SC-39]]'
