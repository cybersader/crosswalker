The registry catalogs the institutional actors and key standards in the ontology ecosystem. Each entry documents what the entity does, what they own/maintain, and how they fit into the [operational landscape](/crosswalker/concepts/operational-landscape/).

> This registry will grow into the **community marketplace** for framework configs — each entity page will eventually link to available [FrameworkConfig](/crosswalker/agent-context/config-schema-design/) presets and [evolution profiles](/crosswalker/agent-context/zz-log/2026-04-03-evolution-pattern-taxonomy-draft/).

## Standard bodies

- [NIST](/crosswalker/reference/registry/nist/) — 800-53, CSF, OSCAL, RMF, OLIR program
- [MITRE](/crosswalker/reference/registry/mitre/) — ATT&CK, D3FEND, ENGAGE, CTID
- [ISO](/crosswalker/reference/registry/iso/) — 27001, 27002, 31000
- [CIS](/crosswalker/reference/registry/cis/) — Controls v8, Benchmarks

## Mapping organizations

- [SCF](/crosswalker/reference/registry/scf/) — 175+ framework hub via STRM (the interlingua)
- [OLIR](/crosswalker/reference/registry/olir/) — NIST's submission-based crosswalk registry
- [CTID](/crosswalker/reference/registry/ctid/) — ATT&CK ↔ 800-53 mappings (MITRE Engenuity)

## Industry groups

- [CRI](/crosswalker/reference/registry/cri/) — CRI Profile for financial sector
- [OWASP](/crosswalker/reference/registry/owasp/) — Top 10, SAMM, ASVS (application security)

## Regulators

- [FFIEC](/crosswalker/reference/registry/ffiec/) — US financial institution examinations
- [SEC](/crosswalker/reference/registry/sec/) — US securities cyber disclosure

## Standards & formats

- [OSCAL](/crosswalker/reference/registry/oscal/) — Machine-readable security controls (NIST)
