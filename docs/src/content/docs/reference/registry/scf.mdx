## Role

**Mapping organization** (pivot hub) — maintains crosswalks between 175+ frameworks through a single central mapping, enabling the [interlingua approach](/crosswalker/concepts/schema-matching/#the-interlingua--pivot-approach).

## How it works

When a framework updates, SCF updates ONE mapping (framework ↔ SCF). All other crosswalks through SCF update automatically. This solves the [N² mapping problem](/crosswalker/concepts/ontology-evolution/#the-translation-analogy).

## Key outputs

- [STRM Bundle](https://securecontrolsframework.com/strm-bundle/) — the complete crosswalk dataset
- SCF control catalog — the pivot ontology itself

## In Crosswalker

SCF is the model for Crosswalker's own [community registry](/crosswalker/agent-context/zz-log/2026-04-03-vision-alignment-decisions/#8-community-marketplace-research-item). The [operational landscape](/crosswalker/concepts/operational-landscape/) shows SCF's role in crosswalk maintenance.
