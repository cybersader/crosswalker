Crosswalker is a **meta-system for [ontology lifecycle](/crosswalker/concepts/ontology-lifecycle/) management**, not just a framework importer. The roadmap reflects this — architecture decisions come first, features are built on that foundation, and every significant decision is [logged with rationale](/crosswalker/agent-context/zz-log/).

See [vision alignment decisions](/crosswalker/agent-context/zz-log/2026-04-03-vision-alignment-decisions/) for the philosophical foundations.

## Current phase: Foundation

## Foundation — "Get the architecture right"

Architecture decisions before more code. The three interconnected schemas (_crosswalker metadata, [FrameworkConfig](/crosswalker/agent-context/config-schema-design/), EvolutionPattern) must be designed together — getting any wrong is expensive to fix later. [Decision: architecture first](/crosswalker/agent-context/zz-log/2026-04-03-vision-alignment-decisions/#4-architecture-decisions-before-more-ui-code)

**Research & spec** (iterative, philosophical, first-principled):

- [ ] **EvolutionPattern taxonomy** — classify how ontologies evolve: release cadence, breaking change policy, ID stability, changelog format, crosswalk maintenance model. **Standalone spec**, not plugin-specific — other tools could implement it. [Research: nothing like this exists](/crosswalker/agent-context/zz-log/2026-04-03-deep-research-synthesis/#research-1-ontology-lifecycle-management--does-anything-solve-this)
- [ ] **FrameworkConfig v2 schema** — evolution metadata, sheet selection, custom transforms, crosswalk definitions. Config-as-code format (JSON/YAML) as [source of truth](/crosswalker/agent-context/zz-log/2026-04-03-vision-alignment-decisions/#2-config-as-code-is-the-source-of-truth); UI edits config files. [Current schema design](/crosswalker/agent-context/config-schema-design/)
- [ ] **_crosswalker metadata v2** — version tracking, source hash, status field, evolution profile reference, migration history. Progressive [metadata tiers](/crosswalker/agent-context/constraint-enforcement/#metadata-tiers) (minimal → standard → extended → temporal)
- [ ] **Migration strategy matrix** — given evolution pattern + old/new version → recommended [SCD type](/crosswalker/concepts/ontology-evolution/#how-data-warehouses-handle-this-slowly-changing-dimensions) + handling strategy
- [ ] **As-code data format research** — define primitives for representing configs, evolution profiles, and migration rules as version-controllable files
- [ ] **Layered architecture design** — spec (universal) → library/SDK (implementations) → integrations (Obsidian plugin, CLI, framework drivers). [Architecture vision](/crosswalker/agent-context/zz-log/2026-04-03-layered-architecture-vision/)
- [ ] **Distribution architecture** — pnpm monorepo: `@crosswalker/core` (pure logic) + plugin (Obsidian wrapper) + CLI (Node.js `fs` wrapper). VaultAdapter interface bridges platforms. [Research](/crosswalker/agent-context/zz-log/2026-04-03-distribution-architecture-research/)
- [ ] **Monorepo restructure** — extract ~50-60% of current codebase into `packages/core/`, keep plugin UI in `packages/plugin/`, create `packages/cli/`. After spec work, not before. [Sequence rationale](/crosswalker/agent-context/zz-log/2026-04-03-distribution-architecture-research/#decision-when-to-restructure)
- [ ] **[Transform engine](/crosswalker/design/transformation/)** — custom build (14 engines evaluated, none fit). 24 transform types, ChunkyCSV-style operations, optional [Arquero](https://github.com/uwdata/arquero)/[JSONata](https://jsonata.org/) escape hatches. [Ontology lifecycle: Import](/crosswalker/concepts/ontology-lifecycle/#import) · [Research](/crosswalker/agent-context/zz-log/2026-04-03-transform-engine-research/)

**Infrastructure:**

- [ ] **PII scanning in CI/CD** — automated detection of personal paths, secrets, API keys before push
- [ ] Bug fixes and polish on current MVP
- [ ] Expand E2E smoke tests minimally (don't test wizard UI — [it will change](/crosswalker/agent-context/zz-log/2026-04-03-vision-alignment-decisions/#6-e2e-testing-after-architecture))

## Formats — "Import anything, transform it properly"

Built on the Foundation spec. The wizard UI is redesigned around FrameworkConfig v2.

- [ ] **Complete [import wizard](/crosswalker/features/import-wizard/) UI** — steps 2-4 redesigned around the new config schema
- [ ] **Complete settings tab** — all 8 sections reflecting new schema
- [ ] **Complete [config browser](/crosswalker/features/config-system/) modal** — search, select, delete, import/export
- [ ] **XLSX parser** — integrate xlsx package into wizard with sheet selector
- [ ] **JSON/JSONL parser** — structured data import
- [ ] **[Transform system](/crosswalker/design/transformation/)** — implement 20+ transform types defined in schema (string, array, type conversion, conditional). [Current types](/crosswalker/design/transformation/)
- [ ] **Improved preview** — expandable folder tree, full note preview in Step 3
- [ ] **E2E test suite** — full import wizard workflow tests, [built from spec](/crosswalker/agent-context/zz-log/2026-04-03-deep-research-synthesis/#research-3-e2e-testing-with-wdio-obsidian)

## Crosswalks — "Link frameworks to each other and to evidence"

The [evidence mapping](/crosswalker/getting-started/grc-teams/) workflow — the core value proposition for GRC teams.

- [ ] **Cross-framework linking engine** — generate typed WikiLinks between imported frameworks using crosswalk data. [Crosswalk specs](/crosswalker/agent-context/framework-crosswalks/)
- [ ] **Link insertion commands** — "Insert framework link" with search modal and metadata form. [Link metadata system](/crosswalker/agent-context/link-metadata-system/)
- [ ] **Batch re-import** — update existing framework folders with [version awareness](/crosswalker/concepts/framework-versioning/) and diff preview. [Resilience patterns](/crosswalker/agent-context/data-model-resilience/)
- [ ] **[FrameworkConfig](/crosswalker/agent-context/config-schema-design/) presets** — per-framework configs with sheet selection, custom transforms, column overrides. Community-contributed via the as-code format.
- [ ] **CLI implementation** — headless import, crosswalk generation, batch operations

## Evolution — "The meta-system"

Implement the [ontology evolution](/crosswalker/concepts/ontology-evolution/) classification system designed in Foundation. This is the novel contribution — [no existing tool solves this](/crosswalker/agent-context/zz-log/2026-04-03-deep-research-synthesis/#research-1-ontology-lifecycle-management--does-anything-solve-this).

- [ ] **Progressive classification UX** — [community pre-classified → guided wizard → auto-detect](/crosswalker/agent-context/zz-log/2026-04-03-vision-alignment-decisions/#1-meta-layer-ux-progressive-classification)
- [ ] **Evolution profile registry** — YAML/JSON per framework, community-contributed
- [ ] **Migration strategy engine** — old version + new version + user evidence → suggested handling + diff preview
- [ ] **Stale [crosswalk](/crosswalker/agent-context/framework-crosswalks/) detection** — surface when imported frameworks have newer versions available
- [ ] **Community marketplace research** — [Obsidian ecosystem constraints](/crosswalker/agent-context/zz-log/2026-04-03-vision-alignment-decisions/#8-community-marketplace-research-item), distribution model options

## Community — "Share and scale"

Build the ecosystem around the spec.

- [ ] **Community config registry** — start with GitHub repo, evaluate built-in registry later
- [ ] **[OSCAL](/crosswalker/concepts/framework-standards/) export** — export to [Open Security Controls Assessment Language](https://pages.nist.gov/OSCAL/) for GRC tool integration
- [ ] **Compliance dashboards** — [Bases views](/crosswalker/concepts/metadata-ecosystem/) for gap analysis, coverage tracking
- [ ] **Report generation** — exportable compliance reports for auditors
- [ ] **Community plugin submission** — publish to Obsidian plugin registry
- [ ] **Spec publication** — publish EvolutionPattern taxonomy as standalone specification
