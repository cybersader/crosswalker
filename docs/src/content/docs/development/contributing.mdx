## Getting started

1. Fork the repository on GitHub
2. Clone your fork and set up the [development environment](/crosswalker/development/setup/)
3. Create a feature branch: `git checkout -b feature/my-feature`
4. Make your changes
5. Run tests: `bun run test && bun run lint`
6. Commit and push
7. Open a pull request

## Code style

- TypeScript with strict types
- ESLint with `eslint-plugin-obsidianmd` rules
- Sentence case for all UI text (Obsidian community plugin requirement)
- Use `Setting.setHeading()` instead of `createEl('h3')` for settings headings

## Commit messages

Keep commit messages concise and descriptive. Focus on the "why" rather than the "what."

## Adding a new parser

To add support for a new file format:

1. Create `src/import/parsers/<format>-parser.ts`
2. Export a `parse<Format>(content, options)` function returning `ParsedData`
3. Add the format to the file picker in `import-wizard.ts`
4. Add unit tests in `tests/<format>-parser.test.ts`

## Editing the documentation

The docs site is built with [Astro Starlight](https://starlight.astro.build/) and lives in the `docs/` folder.

### Quick edits

Every docs page has an **"Edit page"** link in the sidebar. Click it to edit the page directly on GitHub and submit a pull request — no local setup needed.

### Local development

For larger changes or new pages:

```bash
git clone https://github.com/cybersader/Crosswalker.git
cd Crosswalker/docs
bun install
bun run dev        # Preview at http://localhost:4321/crosswalker/
```

Edit files in `docs/src/content/docs/`. Changes hot-reload in the browser.

### File format

Docs use `.mdx` (Markdown with JSX components). Standard Markdown works — you only need JSX for Starlight components like `<Card>`, `<Tabs>`, `<Steps>`:

```mdx
import { Card, CardGrid } from '@astrojs/starlight/components';

<CardGrid>
  <Card title="My card">Content here.</Card>
</CardGrid>
```

### Not an Obsidian vault

The `docs/` folder is **not** an Obsidian vault — the content files are `.mdx`, not `.md`. For rich editing, use VS Code with the [MDX extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx).

### The log pattern

If you have notes, ideas, research, or context that doesn't fit an existing page, add it to the **[Log](/crosswalker/agent-context/zz-log/)** section:

1. Create `docs/src/content/docs/log/YYYY-MM-DD-topic-slug.mdx`
2. Add frontmatter with title, description, and tags
3. Write your notes — doesn't need to be polished

The log is chronological, tagged, and permanent. See the [log index](/crosswalker/agent-context/zz-log/) for the full pattern.

### Testing docs changes

```bash
bun run build          # Verify all pages compile
bun run test:local     # Run Playwright smoke tests
```

## Reporting issues

File issues at [github.com/cybersader/crosswalker/issues](https://github.com/cybersader/crosswalker/issues) with:
- Steps to reproduce
- Expected vs. actual behavior
- Sample data (if possible, anonymized)
- Obsidian version and OS
