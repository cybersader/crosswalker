## The assignment

The project claims [no existing tool solves the ontology evolution problem](/crosswalker/agent-context/zz-log/2026-04-03-deep-research-synthesis/#research-1-ontology-lifecycle-management--does-anything-solve-this). **Prove or disprove this with a thorough competitive analysis as of 2026.**

## What to investigate

1. **GRC platforms:** CISO Assistant, Hyperproof, Vanta, Drata, Anecdotes, ServiceNow GRC, Archer, ZenGRC. Do any of them handle framework versioning and crosswalk maintenance? How?

2. **Ontology management tools:** Protégé, TopBraid, PoolParty, Semaphore, Synaptica. These manage ontologies professionally — do any handle evolution?

3. **Compliance-as-code:** Trestle (IBM), Auditree, InSpec, Open Policy Agent. Do they solve the crosswalk/evolution problem differently (through code rather than UI)?

4. **Obsidian-based approaches:** Are there other Obsidian plugins or vault templates that attempt structured framework management? Check the plugin registry and community forums.

5. **AI-native approaches:** Are there 2025-2026 startups using LLMs to auto-generate crosswalks, detect framework changes, or manage compliance knowledge graphs?

6. **What's Crosswalker's ACTUAL moat?** After finding all competitors, what does Crosswalker do that NONE of them do? Is the [EvolutionPattern](/crosswalker/agent-context/zz-log/2026-04-03-evolution-pattern-taxonomy-draft/) concept truly novel? Is the [file-first philosophy](/crosswalker/agent-context/zz-log/2026-04-03-why-obsidian-why-files/) a differentiator or a limitation?
